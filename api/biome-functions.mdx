## Overview

Biome functions classify cells into ecological zones (ocean, desert, forest, tundra, etc.) based on temperature, moisture, and elevation. They compute blended heat capacity values using smooth hyperbolic tangent transitions to avoid discontinuities in the ODE solver. The system supports 12 distinct biome types with realistic thermal properties.

---

## Temperature Zone Weights

These functions compute smooth weights for temperature-based climate zones using tanh transitions.

### compute_ice_zone_weight

Compute weight for ice sheet zone (T < -10°C).

```julia
compute_ice_zone_weight(T::Real) -> Float64
```

<ParamField path="T" type="Real" required>
  Temperature in Kelvin
</ParamField>

<ResponseField name="weight" type="Float64">
  Weight for ice zone (0.0 to 1.0)
</ResponseField>

**Implementation Details**

- Weight approaches 1.0 when T < 263 K (-10°C)
- Weight approaches 0.0 when T > 263 K
- `BIOME_T_ICE = 263.15` K (ice threshold)
- `BIOME_T_WIDTH = 5.0` K (transition smoothness)
- Formula: `0.5 * (1 - tanh((T - 263.15) / 5.0))`

**Example**

```julia
w = compute_ice_zone_weight(250.0)  # ~0.95 (very cold)
w = compute_ice_zone_weight(263.15) # 0.5 (transition center)
w = compute_ice_zone_weight(280.0)  # ~0.05 (warm)
```

### compute_tundra_zone_weight

Compute weight for tundra zone (-10°C ≤ T < 0°C).

```julia
compute_tundra_zone_weight(T::Real) -> Float64
```

<ParamField path="T" type="Real" required>
  Temperature in Kelvin
</ParamField>

<ResponseField name="weight" type="Float64">
  Weight for tundra zone (0.0 to 1.0)
</ResponseField>

**Implementation Details**

- Combines lower bound (above ice) and upper bound (below cold)
- `BIOME_T_TUNDRA = 273.15` K (0°C)
- Maximum weight ~1.0 occurs around 268 K (-5°C)
- Formula: `lower * upper` where both are tanh-based

**Example**

```julia
w = compute_tundra_zone_weight(268.0)  # ~0.98 (peak tundra)
w = compute_tundra_zone_weight(273.15) # ~0.50
w = compute_tundra_zone_weight(250.0)  # ~0.05 (too cold)
```

## Heat Capacity Blending

### blend_cold_zone_heat_capacity

Blend heat capacity for cold zone between boreal forest and cold steppe.

```julia
blend_cold_zone_heat_capacity(wet_weight::Real) -> Float64
```

<ParamField path="wet_weight" type="Real" required>
  Weight for wet conditions (0.0 to 1.0)
</ParamField>

<ResponseField name="heat_capacity" type="Float64">
  Blended heat capacity in J/(m²·K)
</ResponseField>

**Implementation Details**

- Wet → boreal forest (higher heat capacity)
- Dry → cold steppe (lower heat capacity)
- Linear interpolation: `wet_weight * HC_boreal + (1 - wet_weight) * HC_steppe`

---

</ParamField>

<ResponseField name="heat_capacity" type="Float64">

</ResponseField>

### blend_hot_zone_heat_capacity

Blend heat capacity for hot zone between tropical forest, savanna, and desert.

```julia
blend_hot_zone_heat_capacity(M::Real) -> Float64
```

<ParamField path="M" type="Real" required>
  Atmospheric moisture in kg/m²
</ParamField>

<ResponseField name="heat_capacity" type="Float64">
  Blended heat capacity in J/(m²·K)
</ResponseField>

**Implementation Details**

- Three-way blend based on moisture thresholds:
  - Very wet (M > 8.0) → tropical forest
  - Moderate (1.0 < M < 8.0) → savanna
  - Very dry (M < 1.0) → hot desert
- Uses `compute_desert_moisture_weight` and `compute_forest_moisture_weight`
- Savanna weight fills the gap: `max(0, 1 - desert_weight - forest_weight)`

**Example**

```julia
# Rainforest
hc = blend_hot_zone_heat_capacity(15.0)  # High heat capacity

# Savanna
hc = blend_hot_zone_heat_capacity(4.0)   # Medium heat capacity

# Desert
hc = blend_hot_zone_heat_capacity(0.5)   # Low heat capacity
```

### blend_climate_zone_heat_capacity

Blend heat capacity across all temperature zones for a given moisture level.

```julia
blend_climate_zone_heat_capacity(temp_weights, M::Real) -> Float64
```

<ParamField path="temp_weights" type="NamedTuple" required>
  Temperature zone weights from `compute_all_temperature_zone_weights`
</ParamField>

<ParamField path="M" type="Real" required>
  Atmospheric moisture in kg/m²
</ParamField>

<ResponseField name="heat_capacity" type="Float64">

</ResponseField>

**Implementation Details**

- Computes heat capacity for each temperature zone
- Blends using temperature weights

- Cold/temperate/hot zones further blend based on moisture

### compute_biome_blended_heat_capacity

Calculate heat capacity using smooth biome blending.

```julia
compute_biome_blended_heat_capacity(T::Real, M::Real, elevation::Real) -> Float64
```

<ParamField path="T" type="Real" required>
  Temperature in Kelvin
</ParamField>

<ParamField path="M" type="Real" required>
  Atmospheric moisture in kg/m²
</ParamField>

<ParamField path="elevation" type="Real" required>
  Normalized elevation (-1.0 to 1.0)
</ParamField>

<ResponseField name="heat_capacity" type="Float64">
  Blended heat capacity in J/(m²·K)
</ResponseField>

**Implementation Details**

- **Ocean cells** (elevation < 0): Use ocean heat capacity scaled by depth
  - Formula: `HC_ocean * clamp(1 - elevation, 1.0, 2.0)`
  - Deeper ocean → higher heat capacity (more thermal mass)
  
- **Mountains** (elevation > 0.6): Blend towards mountain heat capacity
  - Pure mountain if weight > 0.99
  
- **Wetlands** (0 < elevation < 0.1): Blend wetland with climate-based capacity
  
- **Land cells**: Use climate-based blending (temperature × moisture)

- **Final blend**: `mountain_weight * HC_mountain + (1 - mountain_weight) * HC_land`

**Example**

```julia
# Deep ocean
hc = compute_biome_blended_heat_capacity(285.0, 10.0, -0.8)  # ~8e6 J/(m²·K)

# Tropical rainforest
hc = compute_biome_blended_heat_capacity(298.0, 15.0, 0.3)   # ~5e6 J/(m²·K)

# Hot desert
hc = compute_biome_blended_heat_capacity(305.0, 0.5, 0.2)    # ~2e6 J/(m²·K)

# Mountain peak
hc = compute_biome_blended_heat_capacity(270.0, 2.0, 0.9)    # ~1.5e6 J/(m²·K)

# Ice sheet
hc = compute_biome_blended_heat_capacity(250.0, 0.1, 0.6)    # ~2.5e6 J/(m²·K)
```

## Biome Classification

### classify_cell_biome

Classify a cell into one of 12 biome types for visualization.

```julia
classify_cell_biome(T::Real, M::Real, elevation::Real) -> Int
```

<ParamField path="T" type="Real" required>
  Temperature in Kelvin
</ParamField>

<ParamField path="M" type="Real" required>
  Atmospheric moisture in kg/m²
</ParamField>

<ParamField path="elevation" type="Real" required>
  Normalized elevation
</ParamField>

<ResponseField name="biome_id" type="Int">
  Biome type ID (0-11)
</ResponseField>

**Biome Types**

| ID | Name | Conditions |
|----|------|------------|
| 0 | Ocean | elevation < 0 |
| 1 | Ice Sheet | T < 263 K (-10°C) |
| 2 | Tundra | 263 K ≤ T < 273 K |
| 3 | Cold Steppe | 273 K ≤ T < 283 K, M < 3.0 |
| 4 | Boreal Forest | 273 K ≤ T < 283 K, M ≥ 3.0 |
| 5 | Grassland | 283 K ≤ T < 293 K, M < 5.0 |
| 6 | Temperate Forest | 283 K ≤ T < 293 K, M ≥ 5.0 |
| 7 | Hot Desert | T ≥ 293 K, M < 1.0 |
| 8 | Savanna | T ≥ 293 K, 1.0 ≤ M < 8.0 |
| 9 | Tropical Forest | T ≥ 293 K, M ≥ 8.0 |
| 10 | Mountain | elevation > 0.6 |
| 11 | Wetland | 0 < elevation < 0.1 |

**Implementation Details**

- Uses hard thresholds (not tanh) for discrete classification
- Priority order: Ocean → Mountain → Wetland → Temperature-based
- Returns integer ID for indexing into color maps

**Example**

```julia
# Ocean
biome = classify_cell_biome(285.0, 10.0, -0.5)  # 0 (BIOME_OCEAN)

# Tropical rainforest
biome = classify_cell_biome(298.0, 15.0, 0.3)   # 9 (BIOME_TROPICAL_FOREST)

# Hot desert
biome = classify_cell_biome(305.0, 0.3, 0.2)    # 7 (BIOME_HOT_DESERT)

# Tundra
biome = classify_cell_biome(268.0, 2.0, 0.4)    # 2 (BIOME_TUNDRA)

# Mountain (overrides temperature)
biome = classify_cell_biome(280.0, 5.0, 0.8)    # 10 (BIOME_MOUNTAIN)
```

## Usage in Simulation

Biome functions are critical for realistic thermal dynamics:

```julia
# In the ODE solver
function temperature_derivative(T, M, elevation, lat, ...)
    # Compute heat capacity based on current state
    heat_capacity = compute_biome_blended_heat_capacity(T, M, elevation)
    
    # Energy balance
    Q_net = Q_solar - Q_ir + Q_latent + Q_transport
    
    # Temperature change (heat capacity in denominator)
    dT/dt = Q_net / heat_capacity
    
    # Higher heat capacity → slower temperature changes → more stable climate
end
```

**Physical Effects**

- **Oceans**: High heat capacity → slow seasonal changes → moderate climate
- **Deserts**: Low heat capacity → fast response → extreme daily/seasonal swings
- **Forests**: Medium-high capacity → buffered temperatures
- **Mountains**: Low capacity → rapid cooling at night, heating during day
